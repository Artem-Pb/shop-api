## Проект #1 — REST API интернет-магазина (каталог + корзина)

**Зачем нужен:** Самый частый запрос на бирже — "нужен backend для магазина".
Это якорный проект портфолио, на который будут смотреть все.

**Что делаешь:**

- Сущности: Product, Category, Cart, CartItem, Order

**Что скринишь:**
- Swagger UI с полным списком эндпоинтов
- Postman-коллекция с примерами запросов/ответов

**GitHub структура:**
/src/main/java/.../controller, service, repository, entity, dto
/docker-compose.yml
/README.md (описание эндпоинтов, как запустить)

**Описание для портфолио:**
> Задача: backend для интернет-магазина. REST API с каталогом, корзиной и оформлением заказа.
> Стек: Java 17, Spring Boot 3, PostgreSQL, Docker. Swagger-документация в комплекте.

## Проект #4 — Spring Security: авторизация JWT + роли

**Зачем нужен:** Почти каждый серьёзный заказ требует авторизации.
Показываешь, что умеешь делать безопасно — это +доверие и +чек.

**Что делаешь:**
- Spring Boot + Spring Security + JWT (jjwt библиотека)
- Регистрация / логин → получение токена
- Защищённые эндпоинты (USER vs ADMIN роли)
- Refresh token механизм
- Можно навесить на проект #1 (магазин)

**Что скринишь:**
- Postman: запрос /login → JWT токен
- Запрос к защищённому эндпоинту с токеном и без (401)
- Схема: регистрация → логин → токен → запрос

**Описание для портфолио:**
> Задача: добавить авторизацию к REST API.
> Реализовал JWT-аутентификацию с ролями USER/ADMIN, refresh-токенами.
> Стек: Spring Security, jjwt, Spring Boot 3. GitHub: [ссылка]

## Правила оформления каждого проекта на Kwork

1. **Обложка:** скрин Swagger UI или диалога бота — чистый, светлый фон
2. **Название:** "REST API магазина — Spring Boot + PostgreSQL + Docker"
3. **Описание (3 абзаца):**
   - Задача клиента (зачем это нужно)
   - Что реализовано (список функций)
   - Стек + ссылка на GitHub
4. **GitHub:** README.md обязателен — как запустить, какие эндпоинты
5. **Категория на Kwork:** "Программы на заказ" или "Создание сайтов" (для API)
