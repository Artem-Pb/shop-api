## Правило веток

```bash
# Начинаем проект
git checkout main
git checkout -b project/1-shop-api

# Работаем, коммитим по смыслу
git add src/...
git commit -m "feat: add Product entity and CategoryRepository"

# Финиш — ветка остаётся, main чистый
```

Каждая ветка — это отдельный репозиторий в GitHub (создаём новый repo на каждый проект).
